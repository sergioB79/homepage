Git & GitHub — Guia rápido prático

O mínimo que precisas para criar um repositório, enviar alterações e trabalhar com branches — sem jargão desnecessário.

1) Criar repositório novo no GitHub e enviar a tua pasta

1
Cria o repositório no GitHub (sem Initialize with README para evitar conflitos).
2
No terminal, dentro da tua pasta:
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/<utilizador>/<repo>.git
git push -u origin main

Dica: Se preferires SSH, usa git@github.com:<utilizador>/<repo>.git e adiciona a tua chave em GitHub → Settings → SSH keys.

2) Ligar um projecto já existente ao GitHub

# já tens commits locais, só falta o remoto
git remote add origin https://github.com/<utilizador>/<repo>.git
git branch -M main
git push -u origin main

3) Fluxo do dia-a-dia (editar → commit → enviar)

git status               # ver o que mudou
git add .                # ou: git add ficheiro1 ficheiro2
git commit -m "Descrição curta e clara"
git pull --rebase        # traz do remoto e mantém histórico limpo
git push                 # envia para o GitHub

Mensagens de commit: pensa em “o quê + porquê” (ex.: fix: corrige path do build no Windows).

4) Branches (trabalhar sem partir a main)

# criar e mudar para uma nova branch
git checkout -b feat/nova-pagina

# trabalhar, fazer commits…
git push -u origin feat/nova-pagina   # publica a branch no GitHub

# abrir Pull Request no GitHub e, depois de aprovado:
git checkout main
git pull
git merge --ff-only feat/nova-pagina  # integra de forma linear
git push

Regra simples: 1 tarefa = 1 branch. Evita branches gigantes.

5) .gitignore essencial

# Node
node_modules/
dist/
*.log

# Python
__pycache__/
*.pyc
.venv/

# OS
.DS_Store
Thumbs.db

Adiciona antes do primeiro commit, ou faz um novo commit “chore: add .gitignore”.

6) Autenticação: HTTPS vs SSH

  • HTTPS: usa o teu utilizador e um Personal Access Token como “password”.
  • SSH: login por chave (sem pedir password cada push). Recomendado para uso diário.
# gerar chave SSH moderna
ssh-keygen -t ed25519 -C "o_teu_email"
# copia a chave pública (~/.ssh/id_ed25519.pub) para o GitHub

7) Renomear a branch principal para main

git branch -M main
git push -u origin main
# no GitHub: Settings → Branches → Default branch → main

8) Resolver conflitos de forma tranquila

git pull --rebase         # rebase é mais limpo
# se houver conflitos:
git status
# edita os ficheiros marcados com <<<<<<< e >>>>>>>
git add <ficheiros_resolvidos>
git rebase --continue

# para desistir e voltar atrás:
git rebase --abort

Se já partilhaste commits e não queres reescrever histórico, usa git pull (merge normal).

9) Publicar só uma subpasta (opcional)

# a partir do repo raiz
git subtree split --prefix=caminho/da/subpasta -b temp-push
git push -u origin temp-push:main
git branch -D temp-push

10) Comandos rápidos que salvam o dia

git remote -v                  # ver remotos
git log --oneline --graph -n 10
git restore <ficheiro>         # desfaz alterações não adicionadas
git restore --staged <ficheiro># tira do stage
git commit --amend -m "msg"    # corrige última msg
git config --global core.longpaths true   # Windows: ficheiros longos
Erros comuns → soluções rápidas
  • remote ref main não existe: o remoto está vazio. Faz primeiro um commit e git push -u origin main.
  • refusing to merge unrelated histories: usa git pull --allow-unrelated-histories e resolve conflitos.
  • Updates were rejected: faz git pull --rebase antes do git push.
  • Ficheiro > 100MB: ativa Git LFS (git lfs track "*.psd").
